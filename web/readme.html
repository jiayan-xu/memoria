Memoria

多信号融合记忆系统 · 知识图谱 · 梦境消化 · 衰减管理

📋 项目概述

Memoria 是一个本地记忆管理系统，为 AI Agent 提供持久化记忆存储和检索能力。基于 SQLite + ChromaDB + FTS5 实现三层记忆存储，通过多信号融合搜索（RRF）、梦境消化（Dream）和衰减管理（Decay）自动维护记忆生命周期。

技术栈

  • 存储引擎：SQLite（元数据+全文检索 FTS5）、ChromaDB（768维向量索引）
  • 中文嵌入：shibing624/text2vec-base-chinese（Sentence Transformers，768维）
  • 搜索算法：RRF（Reciprocal Rank Fusion）融合 Keyword (FTS5) + Semantic (ChromaDB) + Temporal + Importance + Category 五路信号
  • Web 框架：FastAPI + uvicorn，MCP 协议对接
  • 图谱可视化：vis-network 4.21
  • 集成方式：MCP stdio / MCP HTTP（Agent Gateway 联邦网关）

核心架构

记忆分层:
  Hot  (高频) → 缓存层，快速响应，有衰减保护
  Warm (温存) → 主存储层，常规检索
  Cold (冷存) → 归档层，仅语义搜索可达

三层衰减:
  基于 recall_count × decay_factor × importance 计算热度
  定期降级: Hot → Warm → Cold，超期清理

梦境消化周期:
  Light (15s) → 快速扫描新会话
  Deep  (60s) → 深度分析提取候选
  REM   (300s) → 关联编织 + 去重入库

搜索信号融合 (RRF):
  S1 = FTS5 Keyword (直接命中)
  S2 = ChromaDB Semantic (768维向量相似度)
  S3 = Temporal (近期 + 重要性 × 衰减系数)
  S4 = Importance (值 × decay_factor 归一化)
  S5 = Category Match (类别命中加成)

多命名空间

ChromaDB 集合和 SQLite 记录均按 namespace 隔离存储，不同项目的记忆互不干扰。默认命名空间 常熟浦发2/工程部/固废监管，MCP 工具统一接受 namespace 参数。

FEATURE 梦境消化系统 v1

实现三层梦境消化周期，自动从会话中提取记忆：

  • Light 阶段 (15s)：快速扫描 JSONL 会话日志，提取带 remember 关键词的显式记忆

  • 状态追踪：.bridge_processed.json 断点续传，不重复处理
2026-06-11
FEATURE 衰减管理系统 + 时间线 API

实现完整的记忆生命周期管理：

  • 衰减计算：score = recall_count × decay_factor × (importance / 3)，阈值驱动 tier 降级
  • 自动降级：Hot→Warm（score < 3）、Warm→Cold（score < 1）、Cold→清理（超 90 天）
  • 衰减保护：Hot 层级有热度保护，高频访问记忆不轻易降级
  • 时间线 API：GET /decay_timeline?days=90 返回衰减事件历史
  • MCP 工具：memoria_memory_decay 支持查看状态和手动触发
