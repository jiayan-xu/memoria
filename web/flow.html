QClaw Bridge

数据源 · 微信对话索引

监控 QClaw 的 JSONL 会话日志文件，每 15 秒轮询增量。剥离 Conversation info 元数据头，提取纯文本对话内容，过滤系统消息和 Pre-compaction flush。单条消息捕获 + pending 持久化，确保不丢数据。

15s 轮询JSONL 解析元数据剥离增量扫描

Reasonix Bridge

数据源 · 开发会话索引

多根目录递归扫描：全局 sessions/ + 项目目录 projects/*/sessions/。支持 JSONL 会话文件和 .ckpt turn 文件。分批处理（50 条/批，批次间 sleep 2s），断点续传（.bridge_processed.json 状态追踪），避免内存溢出。

多根目录分批处理断点续传120s 轮询

gbrain Bridge

数据源 · 知识库同步

双向桥接管道。正向：gbrain list/get → MCP memory_remember，每 5 分钟轮询新页面。支持三种运行模式：--sync（一次性全量）、--daemon（守护轮询）、--watch（手动触发）。当前已同步概念/实体/时间线页面。

5min 轮询双向管道增量同步守护模式

摄入层

核心 · 数据入口

统一的记忆摄入接口。支持 MCP 协议（memory_remember / memory_observe）和 HTTP POST /api/memories 两种方式。自动去重（内容哈希判重），实时更新 FTS5 全文索引，触发 ChromaDB 向量嵌入。

MCP v1.1.0REST APISHA256 去重实时索引

存储层

核心 · 三引擎架构

SQLite：结构化记忆存储（content、category、importance、decay_factor、source、时间戳），轻量零配置。
FTS5：全文搜索引擎，支持中文分词和布尔查询。
ChromaDB：向量数据库，语义相似度检索，为梦境引擎的跨域关联提供基础。

SQLiteFTS5ChromaDB单文件

梦境引擎

核心 · 记忆加工

Light Sleep：每日自动提取主题、生成摘要。
Deep Sleep：按 importance × decay_factor 衰减记忆权重，60 天冷记忆自动清理。
REM Sleep：跨记忆关联发现、模式识别、Warm→Hot 记忆晋升。模拟人脑记忆巩固过程。

衰减算法关联发现记忆晋升自动清理

Web Dashboard

输出 · 用户界面

深色主题 SPA 单页应用，嵌入 Memoria 服务内（:9003/app/）。四个功能 Tab：🔍 搜索（全文+语义混合）、🗂️ 浏览（时间线+tier 筛选+行内编辑）、📊 图谱（vis.js 交互式网络图）、⚙️ 设置（备份/恢复/参数调节）。

SPAvis.js深色主题零构建

REST API

输出 · 程序接口

完整的 CRUD 端点：GET/POST /api/memories、PUT/DELETE /api/memories/:id。数据导入导出（/api/export、/api/import），系统配置（/api/config）。FastAPI 自动生成 Swagger 文档（/docs），调试友好。

Python SDK

输出 · 客户端库

零依赖 Python 客户端（memoria_client.py，8.6KB）。提供 10+ 方法：remember()、search()、recall()、get_memories()、update_memory()、delete_memory()、export_data()、import_data()、stats()。一行代码接入任何 Python 项目。

零依赖8.6KB10+ APIpip-free

MCP 协议

输出 · AI Agent 集成

基于 MCP 2026-07-28 (v1.1.0) 协议。提供 6 个工具：memory_search（语义搜索）、memory_recall（上下文召回）、memory_remember（写入记忆）、memory_observe（会话记录）、memory_close_session（会话关闭）、memory_prefetch（预加载）。任意支持 MCP 的 AI Agent 可直接接入。
